1.1.1 cpu

cpu的四大功能:程序控制、操作控制、时间控制、数据处理

1-1 cpu结构

结构看上去很复杂，但其实不需要记数据流向，只需要记每个部件用来做什么就好嘞。那么首先，让我们给这些部件划分为两部分，即运算器和控制器

  1. 运算器
    • 算术逻辑单元(ALU) : 用于算术运算和逻辑运算
    • 累加寄存器(AC) : 为ALU提供一个工作区，暂存取出的数或运算结果

      运算器当中至少要有一个AC喔

    • 数据缓冲寄存器(DR) : 暂存从内存中读或写的一条指令或一个数据字
    • 状态条件寄存器(PSW) : 保存条件码，分为状态标志和控制标志
  2. 控制器
    • 指令寄存器(IR) : 暂存从内存中取出的指令
    • 程序计数器(PC) : 总是指向下一条指令的地址

      注意区分IR和PC，一个是存放当前正在执行的指令，一个存放下一条将要执行的执行

    • 地址寄存器(AR) : 保存当前CPU访问的内存单元的地址
    • 指令译码器(ID) : 对指令的操作码字段进行分析解释

1.1.2 数据表示

本小节的例子均假设机器字长为8位

机器数可以分为无符号数和带符号数

  1. 无符号数
    • 纯整数 : 小数点在最低有效数值位之后
    • 纯小数 : 小数点在最高有效数值位之前
  2. 带符号数 :
    • 原码 : 第一位是符号位，其余数位为数值位。
      • 纯整数 :
        [+1]原 = 00000001
        [-1]原 = 10000001
      • 纯小数 :
        [+0.5]原 = 0 1000000
        [-0.5]原 = 1 1000000

    空格表示小数点的位置

    • 反码 : 若为正数则与原码，若为负数则将原码数值位全部取反

      • 纯整数 :
        [+1]反 = 00000001
        [-1]反 = 11111110
      • 纯小数 :
        [+0.5]反 = 0 1000000
        [-0.5]反 = 1 0111111

    • 补码 : 若为正数则与原码，若为负数则将反码最末位+1

      • 纯整数 :
        [+1]补 = 00000001
        [-1]补 = 11111111
      • 纯小数 :
        [+0.5]补 = 0 1000000
        [-0.5]补 = 1 1000000
        下面是原码到补码更简单的转换方法，如果只想记住概念可跳过

        将原码从右往左数的第一个1作为标记位，该位左侧的数值位全部取反，右侧（包括该位本身）不变，即可得到补码，例：
        [-86]原 = 11010110
        [-86]补 = 10101010

    • 移码 : 在数X上添加一个偏移量，规定n位机器字长，偏移量为2n-1。在这种情况下，补码符号位取反就能得到移码
      [+1]移 = 10000001
      [-1]移 = 01111111

注意，原码和反码关于0的表示方法有两种，补码和移码关于0的表示仅有一种
